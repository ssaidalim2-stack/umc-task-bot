# ТЗ — Telegram-бот для управления задачами команды

**Версия:** 1.0 (MVP)
**Дата:** 2026-06-25
**Заказчик:** Саидкарим (Entrium AI / SMM-агентство)
**Статус:** черновик к согласованию

---

## 1. Цель и контекст

Внутренний Telegram-бот для команды, который заменяет «задачи в переписке».
Команда ставит задачи прямо в Telegram, бот хранит их, рассылает уведомления
о новых задачах, напоминает о дедлайнах и позволяет менять статус кнопками,
не выходя из мессенджера.

**Источник правды о задачах:** сам бот (его база данных). Задачи создаются
через команды в Telegram, никаких внешних трекеров для MVP не требуется.

**Ключевая ценность:** ничего не теряется, у каждого исполнителя его задачи
перед глазами, дедлайны не «сгорают» молча.

## 2. Роли и доступы

В системе **две независимые характеристики** пользователя:

### 2.1 Уровень доступа (права)
| Доступ | Права |
|--------|-------|
| **Админ** | Создаёт и назначает задачи, видит все задачи, меняет/удаляет любые, добавляет участников и назначает им роли/доступ. |
| **Исполнитель** | Видит **только свои** задачи, меняет их статус, отмечает «Готово». Задачи **не создаёт**. |

Задачи ставят только админы. **Кто админ — назначает владелец (Саидкарим).** Первичный
список админов задаётся в env (`ADMIN_IDS`), далее админ может выдать доступ через бота.

### 2.2 Специализация (роль в команде)
У каждого участника своя роль — для удобного выбора исполнителя и фильтрации:
`маркетолог`, `таргетолог`, `smm`, `видеограф`, `монтажёр`, `дизайнер` (список расширяемый).

### 2.3 Состав команды (на старте)
| Имя | Специализация | Доступ |
|-----|---------------|--------|
| **Саидкарим** | маркетолог, таргетолог (главный по задачам) | Админ |
| **Бобур** | SMM-менеджер (+ доп. задачи) | Исполнитель |
| **Самандар** | видеограф, монтажёр (по части проектов) | Исполнитель |
| **Асрор** | монтажёр (по проектам) | Исполнитель |
| **Владимир** | дизайнер | Исполнитель |

Состав и роли будут расширяться — добавление участников/ролей без правки кода
(через бота и таблицу `members`).

Привязка — по Telegram user ID.

## 3. Функциональные требования (MVP)

### 3.1 Создание задач
- Команда `/newtask` запускает пошаговый диалог (мастер):
  1. Название задачи
  2. Описание (можно пропустить)
  3. Исполнитель (выбор из участников команды кнопками)
  4. Дедлайн (быстрые варианты: сегодня, завтра, +3 дня, своя дата `ДД.ММ.ГГГГ ЧЧ:ММ`)
  5. Приоритет (низкий / обычный / высокий)
- После создания задаче присваивается короткий ID (например `#123`).
- Подтверждение создателю + уведомление исполнителю.

### 3.2 Уведомления о новых задачах
- При создании задачи бот шлёт **только личное сообщение исполнителю** (общий
  чат не используется) с карточкой задачи и кнопками.
- Карточка задачи: `#ID • Название • Исполнитель • Дедлайн • Приоритет • Статус`.
- Язык карточки и уведомления — по выбранному языку получателя (RU/UZ).

### 3.3 Назначение и переназначение исполнителей
- При создании — выбор исполнителя кнопками.
- Команда/кнопка «Переназначить» (доступна админу) — смена исполнителя,
  новый исполнитель получает уведомление.
- Участники команды регистрируются через `/start` (бот запоминает их ID, имя, username).

### 3.4 Статусы и inline-кнопки
- Статусы: `🆕 Новая` → `🔵 В работе` → `✅ Готово` (+ `⏸ На паузе`, `❌ Отменена`).
- Под каждой карточкой inline-кнопки: **«Взять в работу»**, **«Готово»**, **«Подробнее»**.
- Смена статуса меняет карточку на месте (edit message) и логируется.
- При статусе «Готово» — уведомление создателю/админу.

### 3.5 Напоминания о дедлайнах
- Бот через **Cron** регулярно (раз в ~15 мин) проверяет дедлайны и шлёт:
  - напоминание **за 24 часа** до дедлайна;
  - напоминание **за 1 час** до дедлайна;
  - уведомление о **просрочке** (1 раз после наступления дедлайна, если статус ≠ Готово).
- Напоминания идут **лично исполнителю** (в ЛС); о просрочке дублируется админу в ЛС.
- Каждое напоминание отправляется один раз (флаги в БД, чтобы не спамить).

### 3.6 Просмотр задач
- `/mytasks` — мои активные задачи.
- `/tasks` — все задачи (для админа), с фильтром по статусу.
- `/task #ID` — карточка конкретной задачи.

## 5. Технический стек

| Слой | Технология | Почему |
|------|------------|--------|
| Язык / рантайм | **Node.js (TypeScript)** | Совместимо с твоим стеком (Entrium на Next.js/TS). |
| Telegram framework | **grammY** | Современный, лёгкий, отличная типизация, удобный conversations-плагин для мастера `/newtask`. |
| База данных | **Supabase (Postgres)** | Бесплатный тариф, уже есть аккаунт, RLS, удобная панель. |
| Хостинг бота | **Vercel (Serverless, webhook)** | Бесплатно, не «засыпает», уже есть аккаунт. |
| Напоминания | **Vercel Cron** | Бесплатно, без отдельного сервера. Дёргает endpoint проверки дедлайнов. |
| Доступ к БД | `@supabase/supabase-js` (service role на сервере) | — |

**Режим работы:** Telegram **webhook** → Vercel Function `/api/bot`.
Cron Vercel → Vercel Function `/api/cron` (проверка дедлайнов).

## 7. Логика напоминаний (Cron)

Endpoint `/api/cron` (защищён секретом, дёргается Vercel Cron каждые 15 мин):

```
для каждой задачи где status NOT IN (done, cancelled) и deadline IS NOT NULL:
    now = текущее время
    если deadline - now <= 24ч и > 1ч и not reminded_24h:
        отправить «⏰ Дедлайн через 24 часа: #ID Название»
        reminded_24h = true
    если deadline - now <= 1ч и now < deadline и not reminded_1h:
        отправить «🔥 Дедлайн через 1 час: #ID Название»
        reminded_1h = true
    если now >= deadline и not reminded_overdue:
        отправить «❗ Просрочено: #ID Название» (исполнителю + админу)
        reminded_overdue = true
```

## 8.1 Мультиязычность (RU / UZ)

- Интерфейс бота поддерживает **русский и узбекский** языки.
- При `/start` участник выбирает язык; хранится в `members.lang`. Сменить — `/lang`.
- Все тексты бота (команды, карточки, кнопки, напоминания) хранятся в файлах
  локализации (`locales/ru.json`, `locales/uz.json`) — добавление языка без правки логики.
- Уведомление приходит на языке **получателя** (исполнителю — на его языке).
- Часовой пояс для отображения дат — `Asia/Tashkent` (UTC+5).

---

## 9. Безопасность

- Service role key и токен бота — только в env Vercel, не в репозитории.
- Webhook защищён `secret_token` (Telegram проверяет заголовок).
- `/api/cron` защищён `CRON_SECRET`.
- Проверка прав: команды админа доступны только ID из `ADMIN_IDS`.
- Бот реагирует только на зарегистрированных участников (или по приглашению админа).

## 11. Критерии приёмки MVP

- [ ] Участник регистрируется через `/start`.
- [ ] Админ создаёт задачу через `/newtask` с исполнителем и дедлайном.
- [ ] Исполнитель получает личное уведомление о новой задаче.
- [ ] Кнопки «Взять в работу» / «Готово» меняют статус и карточку.
- [ ] За 24ч и за 1ч до дедлайна приходит напоминание.
- [ ] При просрочке приходит уведомление исполнителю и админу.
- [ ] `/mytasks` показывает актуальный список.
- [ ] Бот работает 24/7 без ручного перезапуска.
